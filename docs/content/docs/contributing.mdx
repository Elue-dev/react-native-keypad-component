<Step>

### cd into the Example App

Next, move into the example app directory:

```bash
cd example
```

Then install the example app dependencies:

```bash
bun install
# or
npm install
```

</Step>

<Step>

### Start the Development Server

Run the example app to preview your local changes:

```bash
bun start
# or
npm run start
```

Then scan the QR code provided on the terminal by Expo and run the app on your Android or iOS device.

</Step>

<Step>

### Create a New Branch

Before making changes, create a branch that describes your work:

```bash
git switch -c feat/improve-docs
```

Use conventional naming like `feat/`, `fix/`, `chore/`, or `docs/`.

</Step>

<Step>

### Make Your Changes

Edit the library or documentation as needed.


</Step>

<Step>


### Commit Using Conventional Commits

Follow the [Conventional Commits](https://www.conventionalcommits.org/) format:

```bash
git commit -m "fix: corrected example path issue"
```

</Step>

<Step>

### Push and Open a Pull Request

Push your branch and open a PR to the main branch:

```bash
git push origin feat/improve-docs
```

Then click **Compare & Pull Request** on GitHub.

</Step>
